The left Column/Nav - top

This isn't your normal Navigation. It uses the Masonry script to sort the gallery boxes on the right.
The class names are for styling and the HREF is for sorting.
What a button says doesn't have to be the same as it's class name and HREF.

You can add or remove nav items. If you remove all the nav items, the site will work fine, it just won't sort by category.
If you want to add a new category, give it a new class name and HREF then make sure you add that class to the gallery boxes that fit in the new category.

The 'About Me' category is one I set up special. A sort of contact page. Feel free to add more boxes with the 'about' class and they will all remain when About Me is clicked.

Setting up your gallery - top

The following is the code showing the second thumbnail box on the demo page:

Each 'thumbnail' box looks similar to this. You can copy this block and change the image file names and text to make your own thumbnail boxes.

Here are the important parts.

  1. The main div needs a class of 'box' and 'col' class. The col class can be col1, col2, col3, col4 or col5. These classes set the max-width of the thumb box. Most of the boxes on my demo are set at col2. The alternate layout page has more random widths for the col classes. The col class widthsare set first thing in the stylesheet and look like this:

    You can modify them if you like.

  2. The last class in this case is 'Web' which is used to identify groups of thumbs if you want to style a catagory in a specific manner, more importantly though, Masonry uses that class name to sort the groups..

  3. The first thing in each box is the A tag that wraps the thumbnail. The thumbnail and the image that the A link points to are the same image. The A points to the larger version. The A needs a class of 'pirobox' to call the enlarging script, and a 'zoom' class to show the magnifying class. If you give the A tag a Title, it will show up when Zoomed.

  4. Finally comes the 'Description' div. You can put what you want in this div. This example starts with the image name in an H5 with it's 'category' wrapped in a span. Then a short description. Then an outgoing link with a 'button' class which styles like you see in the demo.

  5. The final important thing to note is that the thumbnail box has a 'category' class of 'web' in this case. All thumb boxes with the same 'category' class will get sorted by the Masonry script when the left nav link is clicked.

  6. Please be aware that you can use a different 'col' class on each box to get interesting layouts. Keep in mind that gallery will expand to fill a persons monitor, so it will look different on every monitor unless you make them all the same 'col' (width).

Style Changer - top

The Style Changer is mostly to give people ideas on what can be done with this template. I am, however, going to leave it in if you want to do anything fun with it.

Keep in mind though that it's not cookie based so if someone changes the style, leaves the page, and returns, the styles will return to default.

The switcher code is in the main.js file. It works by attaching class names to the body tag when a specific link is clicked. You can add more if you like, or change the styles in the stylesheet for the ones already there.

The Demo page comes with a class of 'dark'.That gives you the default dark page. You'll always want the BODY tag to have a class of either 'dark' or 'light'. If you want the site to always be 'dark stripe' or ' light clouds', you could remove the Style Changer and simply add those classes to the BODY of the page permanently.
